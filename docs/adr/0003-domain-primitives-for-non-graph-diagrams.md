# Domain primitives for non-graph diagrams

Non-software prompts should not be forced through the graph authoring model. We decided that `graph` is primary only for graph-flow scenes; domain-shaped scenes use Visual Primitives as their primary authored material, with `graph` optional only when it adds explanatory value. This was chosen over mandatory graph fallback, which would bias physical/scientific/built-world topics back toward generic node cards, and over AI-generated bitmap diagrams, which would be harder to inspect, animate, validate, and revise deterministically.
