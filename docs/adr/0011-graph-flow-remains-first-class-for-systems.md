# Graph-flow remains first-class for systems

Status: Superseded by ADR 0012

Graph-flow remains a first-class Diagram Family for software and system prompts. Nodes and Edges are still the right abstraction for architecture, request/response flows, queues, databases, APIs, service dependencies, event streams, and similar topics. This was chosen because the primitive system is meant to stop forcing non-graph domains into generic node-card diagrams, not to remove graph diagrams where graph structure is semantically correct.
