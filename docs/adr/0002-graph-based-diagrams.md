# Graph-based diagram authoring and two-column retirement

Status: Superseded by ADR 0012

Diagrams are authored as a graph (nodes + edges + a layout strategy) instead of hand-written x/y coordinates in a fixed 700x600 box. The AI describes nodes (label, icon, kind) and edges (source, target, label, animated flag) and picks a `diagramLayout` strategy (`pipeline` | `client-server` | `hub-spoke` | `stack`); the expander computes all coordinates by applying that strategy to the full canvas. Edges with `animated: true` carry a Packet (a path-animated shape) traveling from source to target, replacing the retired two-column request/response flow animation. The two-column layout is removed entirely — single-column with graph diagrams handles all content, including request/response cycles (via the `client-server` strategy) and comparisons. This was chosen over enhancing the coordinate-based `visualElements` (still cramped, still error-prone) and a hybrid coordinate+graph model (doubles the schema surface). Consequences: the `twoColumnLayout.ts` module, `TWO_COLUMN_KEYWORDS`, column geometry, and the 5-act `TIMINGS` catalog are all retired; the `DIAGRAM_GUIDE` prompt and `validateBrief` are rewritten for the graph model.
